# Log Filter

Небольшой проект на C++20 для потоковой фильтрации логов по IPv4-адресам.

## Что делает

Программа читает входной поток построчно, пытается извлечь IPv4-адрес из каждой строки и выводит только те записи, которые проходят через заданный фильтр.

Поддерживаются два типа правил:

- `subnet` — проверка принадлежности подсети в формате CIDR, например `192.168.0.0/24`
- `range` — проверка попадания в диапазон, например `10.0.0.1-10.0.0.255`

Правила верхнего уровня комбинируются через `OR`: запись проходит фильтр, если её IP удовлетворяет хотя бы одному активному правилу.

## Формат входных данных

Ожидаемый формат строки:

```text
IP - message
```

Примеры:

```text
192.168.0.42 - GET /index.html
10.0.0.15 - login failed
```

Некорректные строки должны игнорироваться.

## Пример использования API

```cpp
auto filter = create_filter({
    {"type", "subnet", "value", "192.168.0.0/24"},
    {"type", "range",  "value", "10.0.0.1-10.0.0.255"}
});

process_stream(input, output, filter);
```

## Основные компоненты

- `create_filter(...)` — создаёт объект фильтра из набора правил
- `process_stream(...)` — обрабатывает входной поток и пишет результат в выходной
- `Filter` — основной класс, который хранит правила, проверяет строки и буферизует подходящие записи

## Внутренняя логика

- IPv4-адрес хранится как `uint32_t`
- подсеть проверяется через маску
- диапазон проверяется обычным сравнением чисел
- обработка идёт потоково, без загрузки всего входа в память
- размер буфера ограничен 1000 записями
- число активных правил ограничено 20

## Сборка

### Требования

- CMake 3.22+
- C++20-компилятор

Для запуска тестов дополнительно требуется Conan 2.x для установки Catch2.

### Сборка проекта

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug -DBUILD_TESTING=OFF
cmake --build build
```

Если требуется собрать проект вместе с тестами, предварительно нужно установить зависимости для тестового окружения.

## Тесты

### Установка Conan

```bash
python3 -m pip install conan
```

### Подготовка зависимостей и сборка с тестами

```bash
conan profile detect --force
conan install . --output-folder=build --build=missing -s build_type=Debug
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=build/conan_toolchain.cmake -DCMAKE_BUILD_TYPE=Debug
cmake --build build
```

### Запуск тестов

```bash
ctest --test-dir build --output-on-failure
```
